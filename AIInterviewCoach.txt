2. GÜN

1.LoginRequestDto oluşturuldu:
	kod mantığı:
public class LoginRequestDto
{
    public string Email { get; set; } = string.Empty;
    public string Password { get; set; } = string.Empty;
} şeklinde. Bunu yapma sebebimiz register işleminde Name, Email, Password alınıyordu. Ama loginde Name gerekmiyor bu yüzden yeni DTO oluşturuldu. 
	DTO'nun amacı: Kullanıcıdan gelen veriyi düzenli ve kontrollü şekilde karşılamak.

2.IAuthService içine LoginAsync metodu eklendi:
	Task<string> LoginAsync(LoginRequestDto request);
	IAuthService bizim sözleşmemiz gibi. Yani diyoruz ki Auth işlemleri yapan her servis, register yapabilmeli ve login yapabilmeli.

3.AuthService içine LoginAsync metodu yazıldı:
	Bu kısımda mantık şu önce email veritabanında aranıyor, email yoksa hata döndürülüyor. Eğer veritabanında bu emaile sahip kullanıcı yoksa user boş gelir sistem böyle bir kullanıcı yok der. Ama kullanıcıya direkt Bu email kayıtlı değil demiyoruz bunun yerine Email ya da şifre hatalı diyoruz çünkü kötü niyetli  biri sistemde hangi mailler var bunu öğrenemesin istiyoruz.

4. Kullanıcının girdiği şifreyi hashliyoruz. Veri tabanında register işlemi sırasında hashlanip kaydedilmiş olan şifre ile karşılaştırıyoruz doğruysa giriş başarılı dönüyor (şimdilik ileride JWT token dönecek). 

5.AuthController içine login endpoint eklendi:
[HttpPost("login")]
public async Task<IActionResult> Login(LoginRequestDto request)
{
    var result = await _authService.LoginAsync(request);

    if (result == "Email veya şifre hatalı.")
    {
        return Unauthorized(result); 
// unauthorized: Http tarafında karşılığı 401 Unauthorized Yanlış mail ya da şifre girilirse bu dönüyor.
    }

    return Ok(result); // Ok işlem başarılı demek HTTP karşılığı: 200 OK
}

Bu sayede dış dünyadan login isteği alabilmemiz sağlandı.
